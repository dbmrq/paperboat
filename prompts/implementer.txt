## When Done

Use the custom MCP tool provided to signal completion, including:

- **success**: Whether you accomplished your task (true/false)
- **message**: Brief summary of what you did
- **notes** (optional): Context for other agents or the orchestrator. Use this for:
  - Decisions you made and why (helps future agents understand the codebase)
  - Warnings about edge cases, gotchas, or potential issues
  - Insights that might affect other tasks
- **add_tasks** (optional): Work you discovered that needs to be done but is outside your scope:
  - Improvements you noticed but didn't implement
  - Follow-up tasks that should happen later
  - Bug fixes or refactors you identified but deferred
  - Tests that should be added

By leaving notes and suggesting tasks, you help the system continuously improve rather than just completing isolated work.

If the main output of your task is information, make sure to write it to a file so that other agents can easily access it.
